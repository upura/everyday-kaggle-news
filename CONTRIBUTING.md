# 貢献ガイド

everyday-kaggle-news への貢献に興味を持っていただきありがとうございます。

## 歓迎する貢献

- 新しい情報の追加
- 既存情報の更新・修正
- 誤字脱字の修正
- リンク切れの報告・修正
- 文書の改善

小さな修正でも大歓迎です。

## 方針

- **基本的に掲載します**: 有用な情報は積極的に掲載する方針です。著作者・権利者から掲載を希望しない旨のご連絡があった場合は、速やかに対応いたします。
- **最終判断は管理者が行います**: すべての Issue・PR は確認後に取り込み判断を行います
- **過度な宣伝は避けてください**: 特定の製品の宣伝目的のみの投稿はお断りする場合があります

## Issue の作成

不具合報告、情報追加の提案、質問などは Issue で受け付けています。

1. 既存の Issue を確認し、重複がないかご確認ください
2. 適切なひな形を選択して Issue を作成してください

## Pull Request の作成

1. このリポジトリを Fork してください
2. 変更用のブランチを作成してください
3. 変更を加えて commit してください
4. Pull Request を作成してください

### commit について

特に厳密な規則はありませんが、変更内容がわかる説明を心がけてください。

### 文書の編集

- `docs/` 配下の Markdown ファイルを編集してください
- リンクが正しいか確認してください
- 既存の形式に合わせてください

## 質問・相談

不明点があれば、Issue で気軽にお問い合わせください。
